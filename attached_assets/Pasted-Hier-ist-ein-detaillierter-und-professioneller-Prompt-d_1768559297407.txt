Hier ist ein detaillierter und professioneller Prompt, den du direkt in **Replit** (in den AI-Agenten oder Ghostwriter) kopieren kannst.

Ich habe mich für einen **Python-Tech-Stack mit Streamlit** entschieden.
**Warum?** Streamlit ist spezialisiert auf Daten-Dashboards. Es ist viel einfacher und schneller als eine normale Webseite, um interaktive Diagramme, Filter und Zeitleisten zu erstellen, die weit über das hinausgehen, was NextDNS standardmäßig bietet.

### Der Replit Prompt

Kopiere den folgenden Textblock in Replit:

---

**Project Title:** NextDNS Advanced Analytics Dashboard

**Role:** You are a Senior Data Scientist and Python Developer.

**Task:** Create a web application using **Python** and **Streamlit** that connects to the NextDNS API to fetch, analyze, and visualize DNS query logs. The goal is to provide better insights, more interactive filtering, and deeper analytics than the standard NextDNS dashboard.

**API Documentation Reference:** [https://nextdns.github.io/api/](https://nextdns.github.io/api/)
**Base URL:** `https://api.nextdns.io`

**Key Features & Requirements:**

1. **Authentication & Setup (Sidebar):**
* Input fields for `API Key` and `Profile ID`.
* Button to "Fetch Data".
* Cache the data to prevent unnecessary API calls during interaction.


2. **Data Fetching:**
* Use the endpoint: `GET /profiles/{profile_id}/logs`
* Implement pagination or a limit selector (e.g., fetch last 1000, 5000, or 10,000 logs) to get a substantial dataset for analysis.
* Handle API errors gracefully (e.g., invalid key).


3. **Data Processing (Pandas):**
* Convert timestamps to the local timezone.
* Categorize queries into "Blocked" vs. "Allowed".
* Extract metadata (Device Name, Protocol, Root Domain).


4. **Advanced Dashboard (The "Better than Original" part):**
* **KPI Row:** Total Queries, Block Rate (%), Top Device, Top Blocked Domain.
* **Time-Series Analysis:** Instead of a static chart, create an interactive timeline (Line Chart) showing Query Volume over time, split by "Blocked" vs "Allowed".
* **Activity Heatmap:** A heatmap showing "Day of Week" vs. "Hour of Day" to identify when the network is most active.
* **Device Forensics:** A specific section where selecting a "Device Name" filters all charts to show only that device's behavior (Who is contacting tracking servers?).
* **GAFAM & Tracker Analysis:** A breakdown of how many requests go to Big Tech (Google, Apple, Meta, Microsoft) over time.


5. **Interactive Log Explorer:**
* Use `st.dataframe` or `AgGrid` to show the raw logs.
* Allow full-text search and sorting by columns (Timestamp, Domain, Device, Protocol).
* Highlight blocked rows in red and encrypted (DoH/DoT) rows in green.


6. **Export Function:**
* A button to download the currently filtered dataset as a CSV file (useful for external reporting).



**Tech Stack:**

* `streamlit` (Frontend/UI)
* `pandas` (Data manipulation)
* `plotly` (Interactive charts)
* `requests` (API calls)

**UI Design:**

* Use a modern, dark theme to match the NextDNS aesthetic.
* Ensure charts are interactive (zoom, hover tooltips).

---

### Was macht diesen Prompt "besser" als das Original?

1. **Heatmaps:** NextDNS zeigt nur "Verlauf". Dieser Prompt verlangt eine Heatmap (z.B. "Wann ist mein Netzwerk am aktivsten?"), was super ist, um Anomalien zu finden (z.B. Smart Home Geräte, die nachts um 3 Uhr "funken").
2. **Korrelation:** Du kannst im "Device Forensics" Teil sehen, *welches* Gerät *welche* Tracker aufruft. Das Original zeigt oft nur Listen getrennt voneinander.
3. **Export:** Du kannst die Daten endlich als CSV ziehen, um sie z.B. in Excel zu speichern.
4. **Interaktivität:** Mit Plotly-Charts kannst du in Zeiträume reinzoomen, statt nur starre Grafiken zu sehen.
